# Integrações de Terceiros
Em muitos cenários corporativos, talvez seja necessário integrar os dados coletados pelo Print Tracker com outros sistemas, como sua solução ERP (Enterprise Resource Planning). O Print Tracker fornece integrações com várias soluções ERP hoje, com a capacidade de integrar imediatamente com qualquer sistema que suporte relatórios CSV enviados por e-mail. Você pode até criar suas próprias integrações personalizadas usando nossa API REST!

## Sistemas ERP {#erp-systems}
### E-Automate {#e-automate}
Como o E-Automate é o sistema ERP mais popular para usuários de MPS corporativos, desenvolvemos uma integração nativa com o E-Automate que envia automaticamente medidores do Print Tracker com base em suas [agendas de faturamento](./20-configuring-settings.md#schedules). Ele também sincroniza os números de equipamento do E-Automate no Print Tracker para que você possa trabalhar facilmente com os dispositivos entre os dois sistemas.

![](https://www.cdn.printtrackerpro.com/images/documentation/integrations-data-processor-meters.png)

Essa integração é alimentada pelo **Print Tracker Data Processor**, um aplicativo que é instalado no local e é compatível com o E-Automate auto-hospedado ou E-Automate hospedado na nuvem.

### Tigerpaw {#tigerpaw}
O Tigerpaw suporta a ingestão de dados do medidor por meio de arquivos CSV enviados por e-mail. Você pode criar um [relatório personalizado](./reports) para ser enviado por e-mail para a Tigerpaw.

### Miracle Service {#miracle-service}
O Miracle Service suporta a integração de dados do medidor por meio de arquivos CSV enviados por e-mail. Você pode criar um [relatório personalizado](./reports) e [agendá-lo](./reports#scheduling-reports) para ser enviado por e-mail ao Miracle Service.

## Entrega Automática {#auto-fulfillment}
### Supplies Network {#supplies-network}
O Print Tracker tem uma integração perfeita com a Supplies Network, um fornecedor líder de entrega automática de consumíveis. Uma vez que a integração esteja ativada, o Print Tracker enviará automaticamente seus medidores para a Supplies Network, onde eles analisarão os medidores para estimar o melhor momento para enviar suprimentos de substituição.

:::info
Habilitar a integração da Supplies Network requer alguma coordenação entre sua conta do Print Tracker e sua conta da Supplies Network. Entre em contato com nossa equipe de suporte para começar.
<Box mb={2}>
    <Button fullWidth size="lg" onClick={() => window.open("mailto:support@printtrackerpro.com")}>Integrate with Supplies Network</Button>

#### Habilitando a integração com a Supplies Network {#enabling-supplies-network-integration}
Uma vez que a integração inicial tenha sido coordenada entre o Print Tracker e a Supplies Network, você pode configurar o Print Tracker para enviar automaticamente medidores para dispositivos específicos ou até mesmo contas inteiras.

#### Em um dispositivo {#on-a-device}
![](https://www.cdn.printtrackerpro.com/images/documentation/integrations-supplies-network-device.gif)
Para habilitar a integração com a Supplies Network para um dispositivo específico:

1. Usando a barra lateral, clique em **Dispositivos > Visualizar dispositivos**
2. Clique no dispositivo para o qual você deseja habilitar a integração
3. Clique na guia **Configurações**
4. Expanda a seção **Automação**
5. Na seção **Supplies Network**, clique no botão de alternância **Habilitado**

#### Em uma conta {#at-an-entity}
![](https://www.cdn.printtrackerpro.com/images/documentation/integrations-supplies-network-entity.gif)
Quando você habilita a integração da Supplies Network em uma conta, os medidores de todos os dispositivos sob essa conta serão enviados por push para a Supplies Network.

1. Verifique se você selecionou a conta para a qual deseja habilitar a integração
2. Usando a barra lateral, clique em **Admin > Gerenciar Conta**
3. Clique na guia **Configurações**
4. Expanda a seção **Automação **
5. Na seção **Supplies Network**, clique no botão de alternância **Habilitado**

## Preços {#pricing}
As integrações que exigem o uso de uma chave de API, como o [Processador de Dados do Print Tracker](#e-automatizar) ou quaisquer integrações personalizadas usando a [API REST](#rest-api) incorrem em encargos adicionais que aparecerão na sua fatura mensal.

> Chaves de API são cobradas a US$ 50 por 50 GB de transferência de dados, por mês, por conta com pelo menos uma chave de API.

Para entender como as chaves de API são cobradas, vamos explorar alguns exemplos:

| Cenário                                                                                                                                                         | Cobrança                                                                                       |
|------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------|
| Você cria uma chave de API em sua conta de revendedor. Você extrai 10 GB de dados por mês.                                                                       | $50 x 1 conta com pelo menos uma chave de API = **$50/mês**                                |
| Você cria uma chave de API em sua conta de revendedor. Você extrai 51 GB de dados a cada mês.                                                                    | $50 x 1 conta com pelo menos uma chave API * 2 blocos de 50GB de transferência de dados = **$100/mês**|
| Você cria duas chaves de API em sua entidade de revendedor, uma para produção e outra para seus desenvolvedores. Você extrai 10 GB de dados entre as duas chaves de API a cada mês.        | $50 x 1 entidade com pelo menos uma chave de API = **$50/mês**                                |
| Você cria uma chave de API na conta do revendedor e outra chave de API para um dos clientes na conta do cliente. Você extrai 10 GB de dados combinados a cada mês. |$50 x 2 contas com pelo menos uma chave de API = **$100/mês**                                 |
